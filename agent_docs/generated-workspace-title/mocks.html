Generated workspace title — exploration mocks

Four directions for making the prompt the primary input and the title a generated artifact. All variants keep the existing pill row (repo · source · new branch), footer, and ⌘↵ submit. Sample task throughout: Add Google OAuth login to the settings page…

A · Prompt-first, derived title line

The prompt becomes the hero input (autofocus). The title is demoted to a small derived row beneath it — same mental model as the branch name: auto until touched, shuffle to regenerate, edit makes it sticky. Generation runs in-modal (debounced) but never blocks submit; if it hasn't landed by ⌘↵, it finishes post-create.

prompt = hero title mirrors branch-field pattern hybrid generation (in-modal, non-blocking) slash commands in prompt

B · Ghost-text title (keeps the title-first layout)

The big title heading stays at the top — but it's no longer the first input. The sparkle sits right next to the title text and is a toggle: on = auto mode (title tracks the prompt), off = you own it (typing toggles it off). The title text is always normal black — the sparkle alone signals auto vs owned. While generating, the shimmer sweeps the text itself — no skeleton bar — and the sparkle animates. A refresh button beside it regenerates on demand.

layout unchanged sparkle = on/off toggle shimmer over text, no skeleton refresh icon = regenerate needs fast in-modal generation to feel live @-mentions in prompt

C · No title field in the modal

A small "Add title" disclosure covers users who want to name things up front.

Changes Panel — Redesign Variants

Variant explorations for the workspace Changes panel (A–C, plus the combined direction in D), addressing: the space taken by the stacked segmented controls, the centered commit button, the cluttered ⋯ overflow menu, and file-row hover consistency with the vertical nav sidebar. The diff viewer's breadcrumb chrome is kept as-is in every variant. All variants apply the sidebar-style row hover (rounded, inset, translucent) — frozen on one row in each frame, with the selected file shown in the sidebar's active style.

Variant A — Single toolbar row

The most compact direction: everything collapses into one 36px toolbar.

  • Files/Changes/Commits becomes three small icon tabs (active = accent pill, like the nav sidebar).
  • All/Uncommitted becomes a lightweight text toggle on the same row.
  • Commit becomes an icon button with a count badge, far right — tooltip carries the words for first-timers.
  • Menu: items keep their order but get labeled groups; the duplicate “Close” is removed (11 items).

Variant B — View dropdown + commit footer

The switcher gets out of the way entirely; commit anchors to a quiet footer.

  • Files/Changes/Commits collapses into a compact labeled dropdown (“Changes ⌄”), shown open here.
  • All/Uncommitted becomes two small filter chips on the same row.
  • Commit lives in a full-width footer bar — words stay visible for first-time users, regulars ignore it.
  • Menu: rarely-used paths fold into submenus (Copy path ›, Open in ›, Close ›) — 7 top-level items.

Variant C — Tighten, don’t rearrange

The conservative direction: keep today’s structure, shrink and merge it.

  • Both segmented controls stay, but at a smaller size.
  • The commit button drops its dedicated centered row — it becomes a small right-aligned button sharing the scope row (2 slim rows ≈ 62px vs ~96px today).
  • Menu: ruthless trim to the items actually used — everything else moves to the command palette via a trailing “All actions… ⌘K”.

Variant D — Combined (chosen direction)

The direction picked from iteration: the Files/Changes/Commits segmented control stays on the panel section row (not a dropdown), A’s text-toggle scope picker sits on its own row beneath it with no border under it, the commit action is a plain “Commit N changes” button anchored to the panel footer (no icon), and the ⋯ menu uses B’s submenu treatment.

  • Row 1 — panel section row: Files | Changes | Commits segmented control, shrunk and left-aligned (not full width), + add-panel “+”.
  • Row 2 — scope picker, under the section row: selected gets a subtle box, unselected is plain text. No border below.
  • Footer — full-width button reading “Commit 3 changes”; words stay for first-time users.
  • ⋯ menu — Variant B’s treatment: 7 top-level items, Copy path / Open in / Close fold into submenus.

Workspace Grouping — Sculptor sidebar

Groups collect related workspaces into a named, colored unit inside a repo section (repo › group › workspaces). Modeled after Dia's tab groups. Three end-to-end variants of the group chrome, each shown in real app chrome and with its edge states: collapsed group, single-workspace group, CLI-born group, long-name truncation, and the right-click context menu.

Variant A — Colored left rail

The group is a light header row (chevron · color dot · name · count). Its members indent under a colored vertical rail that visually ties them together — the lightest, most "folder-like" read. The color lives in the dot and the rail; the name picks up a tint of it. Collapsed, the rail persists and a stacked status-dot preview hints at the members underneath.

Variant B — Tinted container

The whole group — header and members — sits inside a rounded, accent-tinted card with a faint colored border. This is the most literal read of Dia's group card: the color wraps the members as a container, so a group reads as one object you can drag as a unit. Kept deliberately quiet: no workspace count and no group-level run-state — just the name and its color. A hover "⋯" on the header (and right-click) opens the group menu. Collapsed, the card shrinks to just its header.

Drag & drop

Sidebar drag-and-drop is built separately; this shows how groups participate. A group card is a drop target — dropping a workspace onto it adds it as a member. Members reorder within a group, and dragging one out releases it back to the loose list (never deletes it).

Variant C — Labeled divider

The lightest touch: a group is a thin colored rule with a small uppercase color label; members sit below, indented, each with a tiny color tick. No container, no rail. This adds almost no visual weight, so a repo with many groups still scans as a clean list. Color lives in the label text and the divider line. Collapsed, the divider stays and shows a member count.

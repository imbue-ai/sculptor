Mobile Agent Switcher — exploration

Three directions for the bottom switcher on the mobile Workspace. All three free the header to show just the workspace name (stripe-webhooks) and move the agent identity down to where you switch agents. Dot/segment color encodes per-agent run status (● running, ● idle, ● needs input). The new-agent affordance is kept in each.

Click the dots / segments / bar to feel the interaction. Rejected variants stay in this file as a record.

Refined direction — from your feedback
Variant D

Minimal swipeable name → drawer

The active agent shown as a subtle pill with a visible ⌄ so it clearly reads as a control. Swipe the pill left/right to switch agents; tap it to open the bottom drawer (full list + New agent). Minimal, but discoverable.

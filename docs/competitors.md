# Competitors

It's obvious this space is competitive, but we believe there's also room for a diversity of ideas and products. We've been thinking about this problem for a while now, and in a small way, Sculptor is our contribution to the broader ecosystem. If it inspires an idea in at least one person, that's a win.

## Why should I use Sculptor?

- You like thoughtfully designed interfaces
- You want to use pi-agent alongside Claude Code
- You like open source projects
- You like the vibes we give off
- You think the Sculptor team is cool (it's true)
- You feel like trying something new

## Why would I use Sculptor over any CLI/TUI coding agent?

Sculptor is a desktop application. We've put a lot of thought into what a new developer environment would look like when it relies heavily on agents.

## Why would I use Sculptor over Codex Web, Jules, etc. (remote agents)?

Sculptor runs fully locally. You can self-host it if you want to access it remotely.

## Why would I use Sculptor over Claude Code Desktop, Codex Desktop, Cursor, Antigravity, Conductor, Factory, Devin, etc.?

Sculptor is an open source alternative to these projects. We try to take the best features from the products we love.

## Why would I use Sculptor over Opencode, T3Code, Mux, Superset, Warp, etc.?

While all of these projects are open source, Sculptor uniquely focuses on providing defaults for teams spinning up on agent development.

## Why is X coding agent not listed above?

There's too damn many to keep track of.

## Why are we making another one?

It's fun, and our bosses let us do it. We also think we have some unique ideas that the community would benefit from. More to come soon.

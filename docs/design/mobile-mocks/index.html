Sculptor — Mobile / Responsive Mocks

Interactive design exploration for making Sculptor usable in a narrow window. The headline is the mobile Workspace shell (three exploration variants); the rest are simple desktop pages reflowed to one column. These are visual mocks only — no production code. Tap inside Variant A to try the live flow (switch tabs, open the agent sheet, approve/reject, open a file diff).

Target width ≈ 390px Breakpoint < 768px Feature set monitor + chat Theme dark · Radix indigo

Variant A — Bottom tab bar · recommended default

The native iOS/Android pattern. A persistent bottom tab bar (Chat · Changes) is always visible, so the two destinations are one thumb-tap apart. A compact top context bar carries a back affordance and the agent switcher (a bottom sheet). The chat input pins directly above the tab bar; the diff opens as a full-screen push.

Interaction model: flat 2-destination tab bar · agent switcher = bottom sheet · diff = full-screen push. First frame is live — tap around.

Variant B — Header segmented control

No bottom bar. The Chat/Changes switch lives in the header as a Radix SegmentedControl — the same primitive already used in Settings and the diff toolbar — so it reuses an existing component instead of inventing a tab bar. With no bottom chrome, the chat input owns the full bottom edge, giving the message stream the most vertical room. The agent switcher is a dropdown anchored under the title.

Interaction model: view switch = header segmented control · agent switcher = dropdown · diff = full-screen push · maximizes content height. Trade-off: top-of-screen switch is a longer thumb reach than a bottom bar.

Variant C — Left drawer + push navigation

The most "app-like" model. A hamburger opens a left drawer that lists every workspace and its agents (mirroring the desktop sidebar), so you can jump anywhere from one place. Chat is the home screen; Changes is reached by pushing a full-screen view from a header pill (stack navigation) rather than living in a persistent tab. A pending action is hoisted into a sticky banner above the input so it can't scroll out of reach.

Interaction model: global nav = left drawer · Changes = pushed screen via header pill · pending approval = sticky banner. Trade-off: denser nav, Changes is one extra tap (no always-visible tab), best when juggling many workspaces.

TopBar — workspace tabs collapse to a menu

On desktop the top bar shows a Home button, a row of workspace tabs, and search/settings/help icons. At mobile width the tab row can't fit, so it collapses: a hamburger opens a drawer of open workspaces (Home & Settings live there too), the current workspace's name takes the center, and the icon cluster folds into a single overflow menu.
